注意事項

您可能會發現某些視頻的輸出比最後加上黑框的輸入長幾毫秒。這樣做的原因是，由於您每次對aac進行編碼時，都會將額外的音頻啟動樣本添加到音頻流的開頭，從而延長了音頻持續時間。（看這個蘋果文件更多細節。）對於HLS輸出，我們調整音頻持續時間以考慮樣本。

最大輪廓

max_aac_profile:String

API版本：V1，V2

上級：輸出

默認： he-aac

有效值： aac-lc，he-aac或he-aac-v2

兼容的工作類型：全部

例子： he-aac-v2

描述：

AAC音頻編解碼器具有多個配置文件。默認情況下，Zencoder將根據比特率和使用的通道數選擇正確的配置文件：如果max-aac-profile設置為“he-aac-v2"，則40kbps以下的立體聲內容將使用HE-AAC v2；否則，將使用HE-AAC v2。如果max-aac-profile設置為“he-aac"，則84kbps以下的立體聲內容將使用HE-AAC；並且AAC-LC將用於更高的比特率內容，或者將max-aac-profile設置為“aac-lc"。 “he-aac"是默認值，這意味著默認情況下，0-84kbps將使用HE-AAC，85kbps將使用AAC-LC。

液相色譜儀（“低複雜度"）是最常見的AAC配置文件，幾乎每個AAC解碼器都支持AAC-LC。AAC-LC允許單聲道和立體聲，採樣率為8khz-96khz，並且具有高比特率（> 300kbps）。當以80kbps以上的速率編碼立體聲內容或高於40kbps的單聲道內容編碼時，或者要支持僅播放AAC-LC的設備（例如Roku或舊iPod）時，請使用AAC-LC。

HE-AAC（“高效率"）是受到廣泛支持的AAC配置文件，通過實施該協議，在低比特率下聽起來比AAC-LC更好光譜帶複製（SBR）。HE-AAC支持16khz-48khz採樣率和16kbps-128kbps比特率的單聲道和立體聲內容。大多數網絡和移動解碼器都支持HE-AAC，包括Flash，HTML5（支持AAC的任何地方），iPhone，iPad，Android等。將HE-AAC用於80以下的立體聲內容或低比特率的單聲道內容。

HE-AAC v2與參數立體聲（PS）到HE-AAC。這意味著HE-AAC v2僅適用於立體聲內容。單聲道內容只能使用HE-AAC。Flash，HTML5和某些移動設備（Windows Phone 7，Android）支持HE-AAC v2。將HE-AAC v2用於40kbps以下的立體聲內容，但不能用於iOS音頻。（較新的iOS設備可以播放HE-AAC v2內容，但會忽略PS信息，從而產生單聲道內容。）

{
  "input": "s3://zencodertesting/test.mov",
  "outputs": [
    {
      "max_aac_profile": "he-aac-v2"
    }
  ]
}

也可以看看： force_aac_profile

力 _ 輪廓

force_aac_profile:String

API版本：V1，V2

上級：輸出

有效值： aac-lc，he-aac或he-aac-v2

兼容的工作類型：全部

例子： he-aac-v2

描述：

強制使用特定的 AAC 設定檔，而不是讓 Zencoder 選擇位元速率的最佳設定檔。
