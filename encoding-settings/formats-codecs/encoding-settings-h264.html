編碼建議

除了在此頁面上找到的設置之外，您可能還希望包括以下常規視頻設置。

  • video_reference_frames
  • video_codec_profile
  • video_codec_level
  • video_bframes

tuning

tuning:String

API版本：V1，V2

上級：輸出

有效值：電影，動畫，顆粒，psnr，ssim，fastdecode或零延遲

兼容的工作類型：全部

例子：電影

描述：

調整輸出視頻以針對特定的內容類型或特定的編碼優先級進行優化。

在後台，這將設置x264 --tune選項。

可能的值：

  • 電影-針對大多數非動畫視頻內容（不僅是故事片）進行了優化。
  • 動畫-針對動畫進行了優化。請注意，大多數3D動畫的行為更像電影，而不像手繪動畫，因此僅將其用於手繪動畫（動漫，經典迪士尼等）。
  • 顆粒-針對具有高顆粒水平的膠片進行了優化。
  • psnr-使用“峰值信噪比"來優化視頻質量。
  • ssim-使用“結構相似性"來優化視頻質量。
  • fastdecode-降低編碼複雜度，以便於解碼。
  • 零延遲-x264將保留幀的內部緩衝區以提高質量；此設置將刪除該緩衝區，但會降低質量。

  "outputs": [
    {
      "tuning": "film"
    }
  ]
}

crf

crf:Integer

API版本：V1，V2

上級：輸出

有效值：1-51

兼容的工作類型：全部

例子：26

描述：

CRF 是位元控制設定，並設定恆定速率因子。這與常量化器編碼密切相關，儘管 CRF 模式通常通過降低「不太重要」幀的量化器質量來實現比常數 Q 編碼更好的壓縮。進一步了解crf。

Zencoder 品質設定會決定指定視訊的適當 CRF。因此，如果您同時設定 [品質] 和 [CRF]，CRF 設定將會覆寫 [品質]。

有效值為 0-51，較低的值則是品質較佳。一般來說，你不應該低於 16; 16 對於大多數文件來說幾乎無損。大約 24 通常看起來相當不錯。大約 35 看起來很壓縮。

{
  "input": "s3://zencodertesting/test.mov",
  "outputs": [
    {
      "crf": "26"
    }
  ]
}
