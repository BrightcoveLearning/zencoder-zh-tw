縮略圖

縮略圖：數組或哈希

API版本：V1，V2

上級：輸出

兼容的工作類型：全部

例子：

  • “縮略圖”：[ {} ]
  • “縮略圖”：{}

描述：

對於每個輸出，您都可以要求一個或多個縮略圖。

每個輸出都可以有自己的縮略圖集，這些縮略圖集具有唯一性尺寸， base_url， 和訪問控制放。

通過在數組中傳遞多個縮略圖哈希，可以為單個輸出創建多組縮略圖。如果在數組中傳遞了縮略圖選項，則標籤必須設置為set。

套裝S3 訪問控制使用與輸出視頻相同的選項。

縮略圖每組限制為100個。如果根據請求的選項將生成更多縮略圖，則將在輸出中添加警告，並且當達到限制時將停止創建縮略圖。

筆記：沒有像縮略圖一樣的工作。僅使用縮略圖指定輸出 也會創造 使用我們的默認轉碼設置的可計費的全長 mp4 並將其上傳到臨時存儲。

{ "input": "s3://zencodertesting/test.mov", "outputs": [ { "thumbnails": [ { "label": "first", "number": 10 } , { "label": "second", "interval_in_frames": 5000 } ] } ] }

數

數字：整數

API版本：V1，V2

上級：輸出 / 縮圖

有效值：正整數

兼容的工作類型：全部

例子：3

描述：

許多要捕獲的縮略圖。Zencoder將在文件的整個持續時間內抓取均勻分佈的縮略圖。因此，如果您索取1張縮略圖，它將位於文件中間。在8分鐘的視頻上指定3個縮略圖會導致大約2、4和6分鐘的縮略圖。

開始 _ 第一個框架

start_at_first_frame：布爾值

API版本：V1，V2

上級：輸出 / 縮圖

默認：錯誤的

有效值：對或錯

兼容的工作類型：全部

例子：真的

描述：

使用時數字用於生成縮略圖的選項，它將開始從第一幀開始拍攝縮略圖。

例如，在使用時，如果您的視頻時長為5秒，而您使用數字選項，這將在0、1、2、3和4處創建縮略圖。通常，它會以0.83s，1.66s，2.49s，3.32s和4.15s的速度抓取縮略圖。

{ "input": "s3://zencodertesting/test.mov", "outputs": [ { "thumbnails": { "number": 5, "start_at_first_frame": true } }]}

間隔

間隔：浮動

API版本：V1，V2

上級：輸出 / 縮圖

兼容的工作類型：全部

例子：60

描述：

以平均間隔拍攝縮圖，以秒為單位。Zencoder將在文件的每N秒返回一個縮略圖。因此，如果您選擇間隔為60，並且輸入文件的長度為12分鐘，則您將獲得12張縮略圖，每分鐘縮略圖一次。

{ "input": "s3://zencodertesting/test.mov", "outputs": [ { "thumbnails": { "interval": 60 } }]}

倍

次數：數組

API版本：V1，V2

上級：輸出 / 縮圖

兼容的工作類型：全部

例子： [0，30，60，90]

描述：

擷取縮圖的次數，以秒為單位。小數有效。因此，將時間設置為[12.5，25]會抓取兩個縮略圖，一個縮略圖在12.5秒，一個縮略圖在25秒。該值必須是非負數的數組。

{ "input": "s3://zencodertesting/test.mov", "outputs": [ { "thumbnails": { "times": [0, 30, 60, 90] } }]}

方面 _ 模式

aspect_mode：字符串

API版本：V1，V2

上級：輸出 / 縮圖

默認：保存

有效值：保存，拉伸，裁剪或填充

兼容的工作類型：全部

例子：軟墊

描述：

創建縮略圖時要使用的寬高比模式。

  • 保存：默認情況下，Zencoder將保留輸出視頻文件的長寬比，因此，如果您提交寬屏內容並要求標準分辨率，則縮略圖文件將保持寬屏長寬比，並且適合要求的大小。
  • 拉緊：當長寬比模式為“拉伸"時，即使縮略圖扭曲了圖像，縮略圖也將完全匹配請求的寬度和高度。
  • 莊稼：此選項告訴Zencoder通過從頂部/底部或左側/右側裁剪像素來“放大"縮略圖以匹配請求的大小。
  • 軟墊：pad選項告訴Zencoder給信箱縮略圖，以使其與請求的幀大小相匹配。使用此選項可保持輸出視頻的長寬比，但始終獲得相同大小的縮略圖。

筆記：由於視頻縮放算法的限制，縮略圖的尺寸將為偶數。例如，如果您請求100x75的縮略圖，則將獲得100x74的縮略圖。

{ "input": "s3://zencodertesting/test.mov", "outputs": [ { "thumbnails": { "aspect_mode": "pad", "width": 200, "height": 200 } }]}

尺寸

大小：字符串

API版本：V1，V2

上級：輸出 / 縮圖

兼容的工作類型：全部

例子：400x300

描述：

縮略圖的目標分辨率，例如“160×120"。如果未提供尺寸，縮略圖將與輸出視頻的尺寸相同。默認情況下，我們將保留長寬比，因此，如果此大小參數的長寬比與影片的長寬比不匹配，則生成的文件可能與該大小不完全匹配。如果將相應的輸出文件的Stretch設置為true，則此縮略圖也將被拉伸以填充分辨率。

基礎網址

base_url:字符串

API版本：V1，V2

上級：輸出 / 縮圖

有效值：有效的S3，雲文件，GCS，FTP，FTPS或SFTP URL（減去文件名）。格式：s3：// [存儲桶名稱] / [路徑]或ftp：// [用戶]：[密碼] @ [ftp.url] / [路徑]

兼容的工作類型：全部

例子：

  • s3：// my-output-bucket /
  • cf：//用戶名：api_key @ container / path /
  • ftp：// user：password@ftp.example.com/path/to

描述：

縮略圖的輸出目標。如果base_url為空白，則將縮略圖存儲在Zencoder S3存儲桶中。Zencoder S3存儲桶中存儲的文件將在24小時後可用。請注意，文件名在輸出之間不是唯一的（例如frame_0000.png），因此目標位置應該是唯一的目錄或鍵前綴，以避免覆蓋文件。

FTP用戶：我們會嘗試從服務器的根目錄進行寫入，因此請為您的URL使用絕對路徑，以確保我們可以成功寫入服務器。

憑證 description:

如果您通過base_url提供憑據，則這些憑據將出現在對請求的請求中出現的url中工作細節，輸出細節， 等等。如果證書與Zencoder一起存儲，它們將不會出現在這些響應的url中。

{ "input": "s3://zencodertesting/test.mov", "outputs": [ { "thumbnails": { "base_url": "s3://my-output-bucket/" } }]}

檔案名稱

文件名：字符串

API版本：V1，V2

上級：輸出 / 縮圖

默認：框架

兼容的工作類型：全部

例子：

{% raw %}
  • 縮略圖_ {{number}}
  • {{number}} _ {{width}} X {{height}} -縮略圖

描述：

{% endraw %}

可以基於多個屬性為縮略圖文件提供文件名。屬性包括數字，填充數字，寬度，高度和尺寸。不需要添加文件擴展名，因為Zencoder將基於“格式"選項添加文件擴展名。

必須使用數字或填充數字，所有其他均為可選。填充編號使用4個點（0000、0001等）。

高度，寬度和大小將基於生成的縮略圖的實際大小。

範例： {{number}} _ {{width}} X {{height}} -縮略圖

有效字符包括字母，數字，破折號，下劃線和內插值。

請注意，文件名中不支持特殊字符，即使它們是 URL 編碼的。

筆記：如果指定文件名，則指定任何文件名字首將被忽略。

{“輸入”：“s3://zencodertesting/test.mov”，“輸出”：[{“縮略圖”：{“文件名”：“{{number}} _ {{width}} X {{height}} -縮略圖" } } ] }

收割機

標題：哈希

API版本：V1，V2

父母：輸出或者縮圖

兼容的工作類型：全部

例子：{ "Content-Type": "binary/octet-stream" }

描述：

HTTP 標頭， 以便在我們上傳縮圖時與縮圖一起傳送。當前在使用S3和Cloud Files時支持此功能。

Zencoder支持設置以下標頭的有限子集：緩存控制，內容處置，內容編碼，內容類型，過期，x-amz-acl，x-amz-存儲類，x-amz服務器端加密和x-amz-meta- * 。

未指定輸出位置時，標題將被忽略。

{ "input": "s3://zencodertesting/test.mov", "outputs": [ { "thumbnails": { "headers": { "Content-Type": "binary/octet-stream", "x-amz-acl": "public-read-write" } } } ] }

平行上傳 _ 限制

parallel_upload_limit：整數

API版本：V1，V2

默認：中三 30，所有其他目的地為 10

有效值：1到30之間的整數

兼容的工作類型：全部

例子：5

描述：

上載縮略圖時最多可同時上傳的數量。

這可能會加快傳輸時間，具體取決於遠程服務器上的帶寬。請注意，更多的連接會給服務器增加負擔。如果您在上傳超時方面遇到麻煩，或者想防止Zencoder在上傳文件時使用過多帶寬，請將其設置為1。

{ "input": "s3://zencodertesting/test.mov", "outputs": [ { "thumbnails": { "base_url": "ftp://ftp.example.com/thumbnails/", "parallel_upload_limit": 5 } }]}
