概述

Zencoder會自動將字幕從輸入傳遞到輸出，以獲取受支持的格式。您還可以指定字幕文件（SCC格式）以在轉碼期間添加到輸出中。

請注意，混合到視頻軌道上的字幕是不是支持的。

字幕支持的輸入格式：

  • 具有字幕軌道為CEA-608格式的MP4 / M4V視頻文件
  • 使用caption_url選項指定的VTT，SCC，DFXP或SAMI文件

字幕支持的輸出格式：

  • MP4 / M4V文件，帶有CEA-608格式的字幕軌道
  • HLS輸出（帶TS格式的H.264視頻的分段輸出）
  • SCC 和 DFXP 格式也支持字幕類型輸出

對於HLS，必須將字幕映射到輸出以獲取適當的清單內字幕。例如：

{ "label": "Captions", "type": "captions", "segmented": true, "public": true, "filename": "captions.m3u8", "caption_url": "PATH_TO_CAPTION_FILE_HERE", "streaming_delivery_format": "hls" } , { "filename": "master_playlist.m3u8", "type": "playlist", "captions": [{ "path": "Captions/captions.m3u8", "language": "en", "name": "english" } ]，“流”：[ { "path": "video/video-1800.m3u8", "source": "video-res1", "audio": "64k_audio" } ] }

筆記：當前，以下限制適用於字幕轉換：

對於DFXP輸入：

  • 所有特殊字符都必須使用UTF8編碼
  • 僅轉換第一種語言（第一個DIV標籤）
  • 不支持動畫，佈局和區域功能
  • 樣式可以內聯或由ID引用，但是僅應用tts：textAlign屬性
  • 嵌套的DIV和P標籤將被忽略
  • SPAN標籤被忽略
  • 可以使用BR標籤插入明確的休息時間
  • 任何時候屏幕上都只能顯示一個字幕（後續的字幕將終止於先前的字幕）

對於SAMI輸入：

  • 僅列出的第一語言被轉換（第一類選擇器樣式）
  • 不支持內聯樣式
  • 不支持與佈局相關的樣式標籤（表格，div等）
  • 可以使用單獨的P標籤插入明確的休息時間

字幕

標題網址：字符串

API版本：V1，V2

上級：輸出

有效值：SCC，DFXP或SAMI字幕文件的有效URL，以包含在輸出中

兼容的工作類型：全部

例子： http://example.com/captions.scc

description:

包含在輸出中的SCC，DFXP或SAMI字幕文件的URL。

Zencoder支持多種輸出視頻格式的字幕：

  • MP4 / M4V文件，帶有CEA-608格式的字幕軌道
  • HLS輸出（帶TS格式的H.264視頻的分段輸出）

 

筆記：多路復用輸出當前不支持字幕。

{ "input": "s3://zencodertesting/test.mov", "outputs": [ { "caption_url": "http://example.com/captions.scc" } ] }

也可以看看： skip_captions和prepare_for_segmenting

跳過字幕

skip_captions：布爾值

API版本：V1，V2

上級：輸出

默認：錯誤的

有效值：對或錯

兼容的工作類型：全部

例子：真的

描述：

請勿將字幕新增或傳遞至輸出檔案。

{ "input": "s3://zencodertesting/test.mov", "outputs": [ { "skip_captions": "true" } ] }

也可以看看： caption_url和prepare_for_segmenting
