概述

上下文感知編碼會根據您提供的指南以及視頻本身的特徵來創建源視頻的多個副本。鑑於來源的特徵，通過創建本質上毫無價值的演繹，這有助於避免轉碼和存儲視頻的成本。

見CAE指南有關設置的其他信息和建議。

視訊轉碼器

video_codecs:Array

API版本：V2

上級：dynamic_profile_options

有效的數組項目值：h264，hevc

默認值：沒有任何

兼容的工作類型：點播

例子：["h264", "hevc"]

描述：

對於混合編解碼器輸出，要建立轉譯的編解碼器陣列。目前，允許的項目值為h264和hevc。

{
    "dynamic_profile_options": {
      "max_renditions": 6,
      "video_codecs" : ["h264", "hevc"'],
      "video_codec_options": {
        "h264": {
          "min_renditions": 2,
          "max_resolution": { "width": 1280, "height": 720 }
        },
        "hevc": {
          "min_renditions": 2,
          "max_resolution": { "width": 1920, "height": 1080 }
        }
      }
    }
  }

視訊轉碼器選項

video_codecs:Object

API版本：V2

上級：dynamic_profile_options

默認值：沒有任何

兼容的工作類型：點播

例子：

"video_codec_options": {
  "h264": {
    "min_renditions": 2,
    "max_resolution": { "width": 1280, "height": 720 }
  },
  "hevc": {
    "min_renditions": 2,
    "max_resolution": { "width": 1920, "height": 1080 }
  }
}

描述：

對於混合編解碼器輸出，指定每個轉碼器的選項的對象。每個屬性都是一個具有以下名稱的對象h264或者hevc，其屬性是該編解碼器的演繹約束集。這些物件的允許屬性如下：

  • min_renditions必需的
  • min_bitrate
  • max_bitrate
  • max_first_rendition_bitrate
  • min_resolution
  • max_resolution
  • max_first_rendition_resolution
注意：這些設置也用於全局級別。當您在全域和每個轉碼器層級使用設定時，值必須相容。例如，在全局級別設置的 max_renditions 必須大於或等於在每個編解碼器級別設置的 min_rendition 值的總和。
{
    "dynamic_profile_options": {
      "max_renditions": 6,
      "video_codecs" : ["h264", "hevc"'],
      "video_codec_options": {
        "h264": {
          "min_renditions": 2,
          "max_resolution": { "width": 1280, "height": 720 }
        },
        "hevc": {
          "min_renditions": 2,
          "max_resolution": { "width": 1920, "height": 1080 }
        }
      }
    }
  }

品質速率 _ 微調

quality_rate_tune:Number

API版本：V2

上級：dynamic_profile_options

有效值： -40.0-40.0

默認值：0

兼容的工作類型：點播

例子：2

描述：

通過設置quality_rate_tune從1到40的正數，CAE配置文件將以更高的比特率為代價，提供更高的視頻質量。類似地，給定一個介於-1到-40之間的數字，可以降低比特率，但以降低視頻質量為代價。

例如，如果設置為-10，則比特率可能會降低約10％。比特率為10時，比特率通常會提高10％左右。這些數字是近似值，因為CAE表示正值quality_rate_tune在確保非常高的視頻質量方面採取了額外的預防措施，並且可能存在其他限制比特率的參數，例如粒度和再現數。

由於典型的內容以及所需的視頻質量與帶寬的差異很大，因此建議嘗試使用不同的值。 

        {
          "input": "s3://zencodertesting/test.mov",
          "generate_dynamic_profile": true,
          "dynamic_profile_options": {
            "min_renditions": 2,
            "max_renditions": 10,
            "quality_rate_tune": 2
          }
        }

其他設置

其他設置
名稱 類型 描述 預設
speed 整數 編碼速度設置。 3
one_pass 布林 強制進行1遍編碼。 false
tuning 字串 特定於編解碼器的調整選項。

clip_length 字串/數字 創建特定長度的剪輯。可以是格式為HH:MM:SS.S，或以十進製表示的剪輯長度（以秒為單位）。 源持續時間

其他設置
名稱 類型 描述 預設

視頻配置設置

如果CAE未生成輸出中必須具有的特定配置，則可能要使用此功能。

這是一個對像數組，描述了CAE應該/必須在其輸出中產生的視頻分辨率和編解碼器設置（請參見上面的例子）。這個數組是可選的。如果省略，則默認分辨率數組（範圍從192x108到內容的分辨率）將用作CAE的默認輸入集。

視頻配置設置
名稱 類型 描述 必需的？
width 整數 視頻幀寬度[像素] 必需的
height 整數 視頻框高度[像素] 必需的
sample_aspect_ratio 整數：整數對 樣本長寬比（例如1：1、4：3） 可選的
frame_rate 編號 視頻幀率[fps] 可選的
video_codec_profile 字串 視頻編解碼器配置文件 可選的
video_codec_level 字串 視頻編解碼器級別 可選的
video_reference_frames 整數 限制參考幀的數量。默認值= 4（對於H.264基線為3）。 可選的
video_bframes 整數 限制B幀數。默認值= 3（對於H.264基線為0）。 可選的
required 布林 將此設置為true將確保在輸出中建議使用配置設置。默認值= false。 可選的
