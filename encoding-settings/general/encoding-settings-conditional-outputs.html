最小尺寸 (_S)

min_size:String

API版本：V2

家長：條件句

兼容的工作類型：全部

例子：

  • 640x480

描述：

如果源的尺寸小於請求的尺寸，則將跳過輸出。

尺寸是通過將光源的高度和寬度乘以並與所提供尺寸的乘積進行比較來計算的。

筆記：由於視頻元數據的性質以及視頻類型之間的不一致，某些源文件在檢查過程中可能會錯誤地讀取其尺寸，從而導致條件被錯誤地應用或忽略。

{
  "input": "s3://zencodertesting/test.mov",
  "outputs": [
    {
      "skip": {
        "min_size": "640x480"
      }
    }
  ]
}

也可以看看： max_size

最小持續時間 _

min_duration:Integer

API版本：V2

家長：條件句

有效值：正整數

兼容的工作類型：全部

例子：

  • 300

描述：

如果源的持續時間短於請求的持續時間，則將跳過輸出。

持續時間必須以秒為單位。如果信號源少於5分鐘，則要跳過輸出，請使用'300'。

      }
    }
  ]
}

最小音訊 _ 位元速率

min_audio_bitrate:Integer

API版本：V2

家長：條件句

有效值：正整數

兼容的工作類型：全部

例子：

  • 128

描述：

如果源的音頻比特率小於指定的比特率（以kbps為單位），則將跳過輸出。

沒有音軌或無法識別的音頻比特率的源文件將忽略此設置。（看require_audio以獲得更多選擇。）如果源音頻比特率小於128 Kbps，要跳過輸出，請使用128。

筆記：由於音頻元數據的性質以及音頻類型之間的不一致，某些源文件在檢查過程中可能無法正確讀取其比特率，從而導致條件被錯誤地應用或忽略。

{
  "input": "s3://zencodertesting/test.mov",
  "outputs": [
    {
      "skip": {
        "min_audio_bitrate": 128
      }
    }
  ]
}

也可以看看：max_audio_bitrate和require_audio

需要音訊 (_R)

require_audio:Boolean

API版本：V2

家長：條件句

默認：錯誤的

有效值：對或錯

兼容的工作類型：全部

例子：true

描述：

如果源文件沒有音軌，則將跳過輸出。

要在源沒有音軌時跳過輸出，請將此選項設置為true。

筆記：由於媒體元數據的性質和媒體類型之間的不一致，某些源文件在檢查過程中可能會錯誤地讀取其元數據，從而導致條件被錯誤地應用或忽略。


  "input": "s3://zencodertesting/test.mov",
  "outputs": [
    {
      "skip": {
        "require_audio": true,
        "min_audio_bitrate": 64
      }
    }
  ]
}

也可以看看：min_audio_bitrate和require_video
