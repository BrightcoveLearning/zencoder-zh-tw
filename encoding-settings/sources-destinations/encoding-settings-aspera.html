概述

Aspera提供了另一種高速文件傳輸協議，以替代FTP和HTTP等協議。

要使用Aspera，請指定一個Aspera url與文件的路徑。

筆記：Zencoder 在視頻編碼服務器上運行一個 Aspera 客戶端，它連接到您的Aspera Enterprise 或 Connect Server , 託管文件。

這與運行FTP服務器以使文件可用以及使用FTP客戶端進行文件傳輸的Zencoder相似。

作為傳輸政策

aspera_transfer_policy:String

API版本：V1，V2

上級：outputs

默認：公平的

有效值：中等或低

兼容的工作類型：全部

例子：低的

描述：

設置將文件傳輸到Aspera服務器時Zencoder如何使用編碼服務器的可用帶寬。

公平的在編碼服務器上所有正在運行的作業之間平均分配帶寬。

低的使用編碼服務器上的任何可用帶寬，但將優先處理其他作業。如果您擔心過多的流量會使服務器超負荷運行，則低值很有用。

注意：Zencoder不支持Aspera中的“高"選項。默認情況下，我們將使用盡可能多的帶寬，但不會比編碼服務器上的所有其他任務優先訪問一項任務的帶寬。

{
  "input": "s3://zencodertesting/test.mov",
  "outputs": [
    {
      "aspera_transfer_policy": "low"
    }
  ]
}

也可以看看：transfer_minimum_rate和transfer_maximum_rate

傳輸速率 (__S)

transfer_minimum_rate:Number

API版本：V1，V2

上級：outputs

默認：1000

有效值：1000-249999

兼容的工作類型：全部

例子：2500

描述：

設置在使用Aspera進行文件傳輸時要使用的傳輸速率下限。這不能保證達到該速率，但是如果帶寬可用，則文件傳輸不會低於該速度。

這必須小於transfer_maximum_rate。

{
  "input": "s3://zencodertesting/test.mov",
  "outputs": [
    {
      "transfer_minimum_rate": 2500
    }
  ]
}

也可以看看： transfer_minimum_rate和aspera_transfer_policy

傳輸最大速率 (_R)

transfer_maximum_rate:String

API版本：V1，V2

上級：outputs

默認：250000

有效值：1001-250000

兼容的工作類型：全部

例子：25000

描述：

設置在使用Aspera進行文件傳輸時使用的傳輸速率上限。這不能保證達到該速率，但是如果帶寬可用，則文件傳輸不會超過該速度。

同時編碼大量作業時，建議設置最大傳輸速率，以避免源服務器和目標服務器的流量過多。

此值必須大於transfer_minimum_rate。

{
  "input": "s3://zencodertesting/test.mov",
  "outputs": [
    {
      "transfer_maximum_rate": 25000
    }
  ]
}

也可以看看： transfer_maximum_rate和aspera_transfer_policy
