音訊增益

audio_gain:Float

API版本：V1，V2

上級：輸出

有效值：-60至60之間的數字

兼容的工作類型：全部

例子：5

描述：

對音頻應用增益量。在應用其他效果後，這會將音頻的音量增加或減少指定的數量。這主要適用於在應用其他效果後調整級別，或對您已經知道其級別的文件進行調整。稍微超過最大音量的聲音會受到輕微限制，以保持保真度，但是超過最大音量的聲音會明顯失真。看audio_normalize自動增加音量而不會失真。以dB為單位指定，正數或負數，最高60dB。

{
  "input": "s3://zencodertesting/test.mov",
  "outputs": [
    {
      "audio_gain": 5
    }
  ]
}

也可以看看： audio_normalize

音訊標準化 (_R)

audio_normalize:Boolean

API版本：V1，V2

上級：輸出

默認：錯誤的

有效值：對或錯

兼容的工作類型：伏特

例子：真的

描述：

將音訊正常化為 0dB。這會在不引起失真的情況下盡可能增加音頻的音量。為幫助確保通過擴展和壓縮效果進行一致的調整，我們對這些效果之前和之後的音量進行了標準化。看audio_pre_normalize和audio_post_normalize僅在這些點之一進行歸一化。 audio_normalize等效於同時指定audio_pre_normalize和audio_post_normalize。

{
  "input": "s3://zencodertesting/test.mov",
  "outputs": [
    {
      "audio_normalize": true
    }
  ]
}

也可以看看： audio_gain， audio_pre_normalize， 和audio_post_normalize

音訊高通

audio_highpass:Integer

API版本：V1，V2

上級：輸出

有效值：以Hz為單位的頻率，從5到24000

兼容的工作類型：全部

例子：10

描述：

對指定頻率（以Hz為單位）的音頻應用高通濾波器。這樣可以防止低於指定閾值的音頻被包含在輸出中。此設置的一個非常普遍的用途是減少音頻的隆隆聲/噪音，例如在錄製過程中卡車在外面行駛的聲音或一些由風或敲打麥克風引起的聲音。也可以通過指定一個低頻（例如10 Hz）的高通濾波器來減少/消除DC-Shift。

{
  "input": "s3://zencodertesting/test.mov",
  "outputs": [
    {
      "audio_highpass": 10
    }
  ]
}

也可以看看： audio_lowpass

音訊通道 (_L)

audio_lowpass:Integer

API版本：V1，V2

上級：輸出

有效值：以Hz為單位的頻率，從5到24000

兼容的工作類型：全部

描述：

  "outputs": [
    {
      "audio_lowpass": 3000
    }
  ]
}

音訊壓縮比

audio_compression_ratio:Float

API版本：V1，V2

上級：輸出

有效值：1.0至30.0

兼容的工作類型：全部

例子：2.5

描述：

通過以高於指定N：1的指定比例減小音量來壓縮音頻的動態範圍audio_compression_threshold。1.0和30.0之間的比率有效。壓縮對於使安靜點和響亮點之間的音頻音量更加一致非常有用。這可用於減少響點（大聲喊叫或門被撞等），減少震顫。與...結合使用audio_gain或audio_normalization，它可以使語音的安靜部分更容易聽見。

{
  "input": "s3://zencodertesting/test.mov",
  "outputs": [
    {
      "audio_compression_ratio": 2.5
    }
  ]
}

也可以看看： audio_compression_threshold

音訊模式

audio_karaoke_mode:Boolean

API版本：V1，V2

上級：輸出

默認：錯誤的

有效值：對或錯

兼容的工作類型：全部

例子：真的

描述：

通過從左側減去右側音頻通道並僅保留差異來對音頻應用“卡拉OK"效果。僅適用於立體聲輸出。

請謹慎使用此功能：

  • 有時會產生失真
  • 有時什麼也沒做
  • 有時會對背景噪音或音樂產生負面影響
{
  "input": "s3://zencodertesting/test.mov",
  "outputs": [
    {
      "audio_karaoke_mode": true
    }
  ]
}
