    音訊處理設定

    本主題包含有關與音頻處理有關的編碼設置的詳細信息。有關其他音頻特定設置，請參見音訊和速率控制。

    音訊增益

    audio_gain:Float

    API 版本：V1，V2

    上級：輸出

    有效值：-60至60之間的數字

    兼容的工作類型：所有

    例：5

    描述：

    對音頻應用增益量。在應用其他效果後，這會將音頻的音量增加或減少指定的數量。這主要適用於在應用其他效果後調整級別，或對您已經知道其級別的文件進行調整。稍微超過最大音量的聲音將受到輕微限制，以保持保真度，但超過最大音量的聲音顯著增加會導致聲音失真。看到audio_normalize自動增加音量而不會失真。以dB為單位，正或負，最高60dB。

    {
  "input": "s3://zencodertesting/test.mov",
  "outputs": [
    {
      "audio_gain": 5
    }
  ]
}

    也可以看看： audio_normalize

    音訊標準化 (_R)

    audio_normalize:Boolean

    API 版本：V1，V2

    上級：輸出

    默認：假

    有效值：對或錯

    兼容的工作類型：伏特

    例：真正

    描述：

    將音頻標準化為0dB。這會盡可能增加音頻的音量，而不會引起失真。為幫助確保通過擴展和壓縮效果進行一致的調整，我們對這些效果之前和之後的音量進行了標準化。看到audio_pre_normalize和audio_post_normalize只在其中之一標準化。 audio_normalize等效於同時指定audio_pre_normalize和audio_post_normalize。

    {
  "input": "s3://zencodertesting/test.mov",
  "outputs": [
    {
      "audio_normalize": true
    }
  ]
}

    也可以看看： audio_gain， audio_pre_normalize和audio_post_normalize

    音訊高通

    audio_highpass:Integer

    API 版本：V1，V2

    上級：輸出

    有效值：以Hz為單位的頻率，從5到24000

    兼容的工作類型：所有

    例：+10

    描述：

    在以Hz為單位的指定頻率上對音頻應用高通濾波器。這樣可以防止低於指定閾值的音頻被包含在輸出中。此設置的一個非常普遍的用途是減少音頻的隆隆聲/噪音，例如在錄製過程中卡車在外面行駛的聲音或一些由風或敲打麥克風引起的聲音。也可以通過指定一個低頻（例如10 Hz）的高通濾波器來減少/消除DC-Shift。

    {
  "input": "s3://zencodertesting/test.mov",
  "outputs": [
    {
      "audio_highpass": 10
    }
  ]
}

    也可以看看： audio_lowpass

    音訊通道 (_L)

    audio_lowpass:Integer

    API 版本：V1，V2

    上級：輸出

    有效值：以Hz為單位的頻率，從5到24000

    兼容的工作類型：所有

    例：+3000

    描述：

    在以Hz為單位的指定頻率上對音頻應用低通濾波器。這樣可以防止輸出中包含高於指定閾值的音頻。這可用於從音頻中剪切出高音調的嘯叫聲，或創建特殊效果，例如“電話”設置（使用audio_highpass 300和audio_lowpass 3000）。

    {
  "input": "s3://zencodertesting/test.mov",
  "outputs": [
    {
      "audio_lowpass": 3000
    }
  ]
}

    也可以看看： audio_highpass

    音訊壓縮比

    audio_compression_ratio:Float

    API 版本：V1，V2

    上級：輸出

    有效值：1.0至30.0

    兼容的工作類型：所有

    例：+2.5

    描述：

    通過以高於指定N：1的指定比例減小音量來壓縮音頻的動態範圍audio_compression_threshold。1.0和30.0之間的比率有效。壓縮對於使安靜點和響亮點之間的音頻音量更加一致非常有用。這可用於減少響點（大聲喊叫或門被撞等），減少震顫。與...結合使用audio_gain或audio_normalization，它可以使語音的安靜部分更容易聽見。

    {
  "input": "s3://zencodertesting/test.mov",
  "outputs": [
    {
      "audio_compression_ratio": 2.5
    }
  ]
}

    也可以看看： audio_compression_threshold

    音訊壓縮閾值

    audio_compression_threshold:Float

    API 版本：V1，V2

    上級：輸出

    默認： -20

    有效值：0到-120之間的負數

    兼容的工作類型：所有

    例： -30

    描述：

    設置閾值，高於該閾值時將應用音頻壓縮audio_compression_ratio已指定。值以dB為單位指定為負數，低至-120。

    {
  "input": "s3://zencodertesting/test.mov",
  "outputs": [
    {
      "audio_compression_threshold": -30
    }
  ]
}

    也可以看看： audio_compression_ratio

    音訊淡出 (_F)

    audio_fade:Float

    API 版本：V1，V2

    上級：輸出

    有效值：正數，最高30.0

    音訊模式

    audio_karaoke_mode:Boolean

    API 版本：V1，V2

    上級：輸出

    默認：假

    有效值：對或錯

    兼容的工作類型：所有

    例：真正

    描述：

    通過從左側減去右側音頻通道並僅保留差異來對音頻應用“卡拉OK”效果。僅適用於立體聲輸出。

    請謹慎使用此功能：

    • 有時會產生失真
    • 有時什麼也沒做
    • 有時會對背景噪音或音樂產生負面影響
    {
  "input": "s3://zencodertesting/test.mov",
  "outputs": [
    {
      "audio_karaoke_mode": true
    }
  ]
}
