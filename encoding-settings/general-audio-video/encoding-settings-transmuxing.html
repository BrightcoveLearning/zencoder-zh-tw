概述

傳輸是更改視頻或音頻文件的格式，同時保留原始文件中部分或全部流的過程。

如今，Zencoder可以將H.264視頻和AAC / MP3音頻軌道從MP4 / MOV格式傳輸到MP4 / TS（包括創建分段的MP4或分段的TS輸出）。但是，要正確地進行多路復用，輸入文件必須滿足許多要求：

視頻

  • MP4 / MOV容器格式
  • H.264編解碼器
  • 沒有使用b幀

聲音的

  • MP4 / MOV容器格式
  • AAC或MP3編解碼器

分段/分段

  • 最大GOP應小於或等於段持續時間
  • 不得使用HE-AAC-V2

其他

  • 大多數編碼選項對多路復用文件沒有影響，因為流沒有被重新編碼

複製視訊

copy_video:Boolean

API版本：V1，V2

上級：輸出

默認：錯誤的

有效值：對或錯

兼容的工作類型：全部

例子：真的

描述：

指定copy_video將從輸入視頻文件中提取視頻軌道，並將其多路復用到最終的輸出文件中。

{
  "input": "s3://zencodertesting/test.mov",
  "outputs": [
    {
      "copy_video": true
    }
  ]
}

複製音訊 (_R)

copy_audio:Boolean

API版本：V1，V2

上級：輸出

默認：錯誤的

有效值：對或錯

兼容的工作類型：全部

例子：真的

描述：

指定copy_audio將從輸入視頻文件中獲取音軌，並將其多路復用到最終的輸出文件中。

{
  "input": "s3://zencodertesting/test.mov",
  "outputs": [
    {
      "copy_audio": true
    }
  ]
}
