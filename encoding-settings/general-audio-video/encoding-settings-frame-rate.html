框架速率

frame_rate:Float

API版本：V1，V2

上級：outputs

默認：原始 (輸入) 影格速率。

兼容的工作類型：全部

例子：25

描述：

要使用的輸出幀速率，以十進制數表示（例如15或24.98）。

我們建議您不要使用此設置-強制更改視頻幀速率會導致視頻質量降低。如果要防止幀頻超過特定水平，請使用max_frame_rate反而。

對於大於HD的分辨率（即1920x1080），不允許使用高於60的幀頻。

{
  "input": "s3://zencodertesting/test.mov",
  "outputs": [
    {
      "frame_rate": 25
    }
  ]
}

也可以看看： max_frame_rate和抽取

最大框架速率

max_frame_rate:Float

API版本：V1，V2

上級：outputs

兼容的工作類型：全部

例子：30

描述：

將輸出幀速率限制為一定數量。如果幀速率否則會超過此數字（例如，輸入幀速率太高），則會將幀速率降低到max_frame_rate環境

示例：如果您設置max_frame_rate到29.97，並且輸入的幀速率為59.94，輸出文件的幀速率為29.97。

我們建議您使用此設置，而不是設置確切的幀率，原因有兩個：

首先，更改frame_rate可能會由於口吃和抖動而導致視覺質量下降。

其次，將幀速率設置為固定數量實際上可能會使輸出視頻的幀速率增加到超過輸入幀速率，這將導致不必要的文件大小增加（或質量降低）。

{
  "input": "s3://zencodertesting/test.mov",
  "outputs": [
    {
      "max_frame_rate": 30
    }
  ]
}

也可以看看：幀率和抽取

min_frame_rate

min_frame_rate:Float

API版本：V1，V2

上級：outputs

兼容的工作類型：全部

例子：30

描述：

將最小輸出幀速率限制為一定數量。如果幀速率否則會小於此數字（例如，輸入幀速率太低），則會將幀速率提高到min_frame_rate環境。

這min_frame_rate價值 必須小於 這max_frame_rate價值。

示例：如果您設置min_frame_rate到30.0，輸入的幀率為29.97，輸出文件的幀率為30.0。

{
  "input": "s3://zencodertesting/test.mov",

破解

decimate:Integer

API版本：V1，V2

上級：outputs

有效值：正整數

兼容的工作類型：全部

例子：2

描述：

將輸入幀速率除以指定的數字。如果您知道要將幀速率降低一半，但又不知道輸入幀速率是多少，這將很有用。在輸入幀率為20的情況下，如果將decimate設置為2，則輸出幀率為10。你仍然可以設置最大幀率如果要確保抽取的幀速率不超過某個值。

輸出視頻的frame_rate等於輸入幀率除以decimate價值。因此，如果輸入為24 fps，並且您將decimate設置為4，則輸出將為6 fps（24/4）。

{
  "input": "s3://zencodertesting/test.mov",
  "outputs": [
    {
      "decimate": 2
    }
  ]
}

關鍵影格間隔 (_L)

keyframe_interval:Integer

API版本：V1，V2

上級：outputs

默認：250

有效值：正整數

兼容的工作類型：全部

例子：100

描述：

設置每個關鍵幀之間的最大幀數。默認情況下，將至少每250幀創建一個關鍵幀。指定不同的關鍵幀間隔將使您可以在視頻中創建更多或更少的關鍵幀。關鍵幀間隔應指定為正整數。例如，值100將每100幀創建一個關鍵幀。

較大數量的關鍵幀將增加輸出文件的大小，但將允許大多數播放器進行更精確的清理。

請注意，這是最大關鍵幀間隔。根據場景轉換和其他因素，可以使用更多關鍵幀來確保高質量編碼。

為了H.264視頻，關鍵幀是IDR幀。

keyframe_interval 優先於關鍵幀率 .

{
  "input": "s3://zencodertesting/test.mov",
  "outputs": [
    {
      "keyframe_interval": 100
    }
  ]
}

也可以看看：關鍵幀率

關鍵影格速率

keyframe_rate:Float

API版本：V1，V2

上級：outputs

默認： keyframe_interval為250

兼容的工作類型：全部

例子：1.5

描述：

設置每秒的關鍵幀數。因此，值0.5將導致每兩秒顯示一個關鍵幀。值為3將導致每秒三個關鍵幀。

默認是通過控制關鍵幀keyframe_interval，默認值為250。根據視頻幀速率，這會導致每8-11秒生成一個關鍵幀。

較大數量的關鍵幀將增加輸出文件的大小，但將允許大多數播放器進行更精確的清理。

keyframe_interval優先於keyframe_rate。

{
  "input": "s3://zencodertesting/test.mov",
  "outputs": [
    {
      "keyframe_rate": 1.5
    }
  ]
}

也可以看看：fixed_keyframe_interval， forced_keyframe_interval， 和force_keyframe_rate

固定關鍵影格間隔

fixed_keyframe_interval:Boolean

API版本：V1，V2

上級：outputs

默認：錯誤的

有效值：對或錯

兼容的工作類型：全部

例子：真的

描述：

默認情況下，指定keyframe_interval僅保證兩個關鍵幀之間的最大幀數。編碼器可以決定在keyframe_interval需要它之前放置一個關鍵幀。例如，可以通過將關鍵幀放置在場景更改處來實現更好的壓縮。將此選項設定為 true 可避免此變異性。

僅當使用H.264，HEVC或VP6 video_codec。

{
  "input": "s3://zencodertesting/test.mov",
  "outputs": [
    {
      "fixed_keyframe_interval": true
    }
  ]
}

也可以看看：keyframe_interval， forced_keyframe_interval， 和force_keyframe_rate

強制關鍵影格間隔

forced_keyframe_interval:Integer

API版本：V1，V2

上級：outputs

有效值：正整數

兼容的工作類型：全部

例子：10

描述：

強制以指定的幀間隔（每N幀）創建關鍵幀，同時仍允許編碼器將關鍵幀放置在其他位置，例如在場景切換時。相對於fixed_keyframe_interval僅將關鍵幀放置在指定的時間間隔。

僅當使用H.264或者HEVCvideo_codec。

{
  "input": "s3://zencodertesting/test.mov",
  "outputs": [
    {
      "forced_keyframe_interval": 10
    }
  ]
}

也可以看看：keyframe_interval，fixed_keyframe_interval， 和force_keyframe_rate

強制關鍵影格速率

forced_keyframe_rate:Floating point number

API版本：V1，V2

上級：outputs

有效值：0.001至100

兼容的工作類型：全部

例子：0.1

描述：

通過考慮幀速率，通過指定每秒的關鍵幀數來設置強制關鍵幀間隔。

如果此選項被忽略，則forced_keyframe_interval已指定。

僅當使用H.264或者HEVCvideo_codec。

{

  "outputs": [
    {
      "forced_keyframe_rate": "0.1"
    }
  ]
}

產生框架索引

generate_frame_index:Boolean

API版本：V2

上級：outputs

默認：錯誤的

有效值：對或錯

兼容的工作類型：全部

例子：真的

描述：

產生一個製表符分隔的文件，該文件具有三個整數值：幀號，幀大小（字節）和最後一個關鍵幀號。每行用空格填充，寬度為42列。

除非輸出包含視頻流，否則不能使用此選項。

默認的索引文件名 '框架 _index.tab ' 可以被覆蓋frame_index_filename。

{
  "input": "s3://zencodertesting/test.mov",
  "outputs": [
    {
      "generate_frame_index": true,
      "frame_index_filename": "my_frame_index.txt"
    }
  ]
}

也可以看看：frame_index_filename
