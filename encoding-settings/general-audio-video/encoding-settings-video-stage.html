video_bit_depth

video_bit_depth:Number

API版本：V2

上級：輸出

默認：10

有效值：8、10或12

兼容的工作類型：現場直播

例子：5

描述：

每個顏色通道的位數。

通常你不會設置它，而是指定video_codec_profile這將提供位深度。

{
  "input": "s3://zencodertesting/test.mov",
  "outputs": [
    {
      "video_bit_depth": 10
    }
  ]
}

視訊參考影格

video_reference_frames:String

API版本：V2

上級：輸出

默認：3

有效值：0-16，auto

兼容的工作類型：現場直播

例子：5

描述：

要使用的參考框架數目。

更多的參考幀會導致更高的壓縮質量，但會增加解碼複雜度。在實踐中，超過 5 很少有很多好處。

部分由速度也video_codec_profile .設置為“自動”以允許我們速度設置自然選擇這個數字。

我們默認為 3 是壓縮和解碼複雜性的一個很好的折衷。

將1用於為舊版iPod或第一代iPhone視頻創建的視頻，或其他受技術限制的解碼器。

{
  "input": "s3://zencodertesting/test.mov",
  "outputs": [
    {
      "video_reference_frames": 5
    }
  ]
}

視訊轉碼器輪廓

video_codec_profile:String

API版本：V2

上級：輸出

默認：H.264：baseline，HEVC：main，VP9：0

有效值：編解碼器相關

兼容的工作類型：全部

例子：高的

描述：

設置視頻編解碼器使用的編碼配置文件。

  • H.264配置文件：baseline，main，high，high10，high422，high444。
  • HEVC配置文件：main，main10，main12，main422-10，main422-12，main444-8，main444-10，main444-12。
  • VP9配置文件：0、1、2、3。
  • AV1配置文件：main，high和professional。默認：main。

通常，較低級別由更廣泛的設備支持，並且可能更易於解碼。更高的級別提供更多的功能，例如額外的位深度和色度二次採樣選擇。在某些情況下，更高的級別還可以提高壓縮效率。

{
  "input": "s3://zencodertesting/test.mov",
  "outputs": [
    {
      "video_codec_profile": "high"
    }
  ]
}

視訊轉碼器層級

video_codec_level:String

API版本：V2

上級：輸出

默認：使用基於輸出視頻的水平圖自動計算。看H.264級， HEVC級別， AV1等級。

有效值：有效級別：H.264等級， HEVC水平， AV1等級。

兼容的工作類型：全部

例子：4.1

描述：

限制比特率宏塊（H.264）或比特率編碼樹單位（HEVC），限制比特率和編碼樹單位（CTU）（AV1）。

主要用於裝置相容性。例如，iPhone支持H.264 3級，這意味著視頻的coder_bitrate_cap不能超過10,000kbps。一般而言，如果您的目標是特定裝置需要此設定，您才應該變更此設定。

{
  "input": "s3://zencodertesting/test.mov",
  "outputs": [
    {
      "video_codec_level": "4.1"
    }
  ]
}

視訊

video_bframes:Integer

API版本：V2

上級：輸出

默認：0

有效值：0-16

兼容的工作類型：全部

例子：2

描述：

連續B幀的最大數量。

使用 B 幀可以以相同的位元速率提高品質，但在 CPU 上的要求更高。這在舊計算機或設備上可能會出現問題，但在現代硬件上通常可以流暢運行。

H.264 由 b 幀支持主要或高調，不是基線。默認值是0，以獲得最大的兼容性。

對於HEVC，可以在任何級別上使用b幀。預設值為3。

為了改善壓縮/品質，我們建議使用 3 的值。高於5或6的值很少會帶來很多好處，並且會大大增加編碼時間。

{
  "input": "s3://zencodertesting/test.mov",
  "outputs": [
    {
      "video_bframes": 2
    }
  ]
}

也可以看看： video_codec_profile

顏色 (中繼資料)

color_metadata:String

API版本：V2

上級：輸出

默認：preserve

有效值：preserve或者discard

兼容的工作類型：全部

例子：discard

描述：

H.264，HEVC和VP9輸出當前支持此選項。如果輸入視頻具有矩陣係數，傳輸特性和顏色原色元數據，則Zencoder將preserve這些設置為默認設置。

為確保此元數據未嵌入輸出中，請設置color_metadata至discard。

{
  "input": "s3://zencodertesting/test.mov",
  "outputs": [
    {
      "color_metadata": "discard"
    }
  ]
}
