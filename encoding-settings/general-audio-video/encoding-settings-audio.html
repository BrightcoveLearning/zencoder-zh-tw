音訊取樣 _ 速率

audio_sample_rate:Integer

API版本：V1，V2

上級：outputs

有效值：有效的採樣率。取決於編解碼器；通常是8000、11025、12000、16000、22050、32000、44100、48000、96000

兼容的工作類型：伏特

例子：44100

描述：

音頻採樣率，以Hz為單位。

我們建議您僅在有充分理由的情況下才使用此設置。強制輸出採樣率可能會導致意外問題-例如，每個audio_codec僅支持有限範圍的採樣率，並且每個採樣率僅與有限範圍的比特率兼容。您不能以16kbps的速率編碼48Khz AAC，也不能以320kbps的速率編碼8000hz MP3。

 

默認情況下，將使用輸入採樣率（即，音頻將不會重新採樣），儘管高於96000的採樣率將降至最大96000。

FLV 視頻格式僅支持 11025、22050 和 44100 的採樣率。默認情況下，我們只會將這些採樣率用於FLV視頻，並將其他採樣率移至與FLV兼容的最接近的採樣率。使能夠strict_mode如果您希望不兼容的採樣率導致失敗的工作，而不是自動更正為有效的sample_rate。

  {
    "input": "s3://zencodertesting/test.mov",
    "outputs": [
      {
        "audio_sample_rate": 44100
      }
    ]
  }

也可以看看：音頻比特率和音頻質量

最大 _ 音訊 _ 取樣 _ 速率

max_audio_sample_rate:Integer

API版本：V1，V2

上級：outputs

有效值：有效的採樣率。取決於編解碼器；通常是8000、11025、12000、16000、22050、32000、44100、48000、96000

兼容的工作類型：居住

例子：44100

描述：

最大音頻採樣率。覆蓋音頻採樣率設置以確保採樣率不超過提供的數字。

如果您輸入的視頻的採樣率為48000，而您的max_audio_sample_rate設置為22050，我們將音頻重新採樣為22050。如果其採樣率為11025，我們將不理會它。

如果目標播放設備不支持更高的採樣率，請使用此選項。

默認情況下，將使用輸入採樣率（即，音頻將不會重新採樣），儘管高於96000的採樣率將降至最大96000。

FLV 視頻格式僅支持 11025、22050 和 44100 的採樣率。默認情況下，我們只會將這些採樣率用於FLV視頻，並將其他採樣率移至與FLV兼容的最接近的採樣率。使能夠strict_mode如果您希望不兼容的採樣率導致失敗的工作，而不是自動更正為有效的sample_rate。

  {
    "input": "s3://zencodertesting/test.mov",
    "outputs": [
      {
        "max_audio_sample_rate": 44100
      }
    ]
  }

也可以看看：音頻採樣率 , 音頻比特率， 和音頻質量

音訊頻道 (_L)

audio_channels:Integer

API版本：V1，V2

上級：outputs

默認：1，如果原始文件是單聲道的; 否則，2.

有效值：1、2 或 6（6 僅當輸入內容在 5.1 配置中有 6 個通道時才有效，或input_audio_channels提供有效的 5.1 配置）

兼容的工作類型：全部

例子：1

描述：

使用的音頻通道數：1（單聲道）或2（立體聲）。

請注意，在檢查時，單聲道AAC音頻有時會錯誤地自我報告為立體聲。我們建議您使用iTunes獲取AAC音頻的真實頻道數。請注意，還支持5.1聲道音頻輸出。為此，您將使用和output_audio_channels反而。

  {
    "input": "s3://zencodertesting/test.mov",
    "outputs": [
      {
        "audio_channels": 1
      }
    ]
  }

也可以看看：音頻比特率 , 音頻質量 , input_audio_channels， 和輸出音頻通道

音訊位元 _ 深度

audio_bit_depth:Integer

API版本：V1，V2

上級：outputs

默認：16

有效值：PCM為16、24、32;其他所有16個。

兼容的工作類型：全部

例子：24

描述：

每個樣本中的位數：十六、二十二

  {
    "input": "s3://zencodertesting/test.mov",
    "outputs": [
      {
        "audio_bit_depth": 24
      }
    ]
  }

音訊語言

audio_language:String

API版本：V1，V2

上級：outputs

有效值：ISO 639-1或639-2語言代碼

兼容的工作類型：全部

例子：恩

描述：

設置輸出的音頻軌道的語言標識符，以描述其語言內容。可以是ISO 639-1或ISO 639-2語言代碼（分別為2個或3個字符）。

注意：當前支持MP4，MKV和WEBM格式輸出以及DASH流傳輸輸出。

  {
    "input": "s3://zencodertesting/test.mov",
    "outputs": [
      {
        "audio_language": "en"
      }
    ]
  }

輸入音訊頻道

input_audio_channels:Hash

API版本：V1，V2

上級：outputs

有效值：頻道標識符和跟踪索引值的哈希。

兼容的工作類型：全部

例子：

  • {"FR":1,"FL":2,"FC":3,"BL":4,"BR":5,"LFE":6}
  • {"DL":7,"DR":8}
  • {"FL":"1:1","FR":"1:2"}

描述：

設置輸入的音頻軌道與其通道標識符之間的映射。此選項可用於獲取具有多個音頻軌道的輸入，並在輸出上創建單個音頻軌道。哈希鍵必須是有效的通道標識符，並且值必須是整數，該整數與輸入上的音軌索引相對應。

通過指定附加通道索引，此選項還可用於從輸入中選擇單個通道。例如，將通道標識符設置為“1：1"將選擇輸入中第一音頻軌道的第一通道。

有效的音頻通道標識符："FL", "FR", "FC", "LFE", "BL", "BR", "DL", "DR".

  {
    "input": "s3://zencodertesting/test.mov",
    "outputs": [
      {
        "input_audio_channels": {
          "FL": 1,
          "FR": 2,
          "FC": 3,
          "LFE": 4,
          "BL": 5,
          "BR": 6
        }
      }
    ]
  }

輸出音訊通道

output_audio_channels:Array or Integer

API版本：V1，V2

上級：outputs

有效值：通道佈局或整數，用於定義輸出音頻通道的數量。

兼容的工作類型：全部

例子：

  • ["FL","FR","FC","BL","BR","LFE"]
  • 2
  • 1
  • 6（6 僅當輸入內容在 5.1 配置中有 6 個通道時才有效，或input_audio_channels提供有效的 5.1 配置）

描述：

將此與input_audio_channels定義自定義輸出音頻通道佈局或混音的選項。

此選項有兩種模式，一種用於指定自定義音頻通道佈局，另一種用於指定單聲道或立體聲混音。

此選項可以是有效通道標識符的數組，也可以是整數。要創建立體聲混音，請將此參數設置為2。要創建單聲道混音，請將此參數設置為1。

有效的音頻通道標識符：“FL"，“FR"，“FC"，“LFE"，“BL"，“BR"，“DL"，“DR"。

通過將此選項設置為有效通道標識符的數組，可以創建自定義音頻通道佈局。

筆記：此選項優先於audio_channels。

  {
    "input": "s3://zencodertesting/test.mov",
    "outputs": [
      {
        "input_audio_channels": {
          "FL": 1,
          "FR": 2,
          "FC": 3,
          "LFE": 4,
          "BL": 5,
          "BR": 6
        },
        "output_audio_channels": [
          "FL",
          "FR",
          "FC",
          "BR",
          "BL",
          "LFE"
        ]
      }
    ]
  }
