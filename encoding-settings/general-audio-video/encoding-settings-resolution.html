概述

您可以使用一種或多種分辨率設置來控制輸出視頻的分辨率。

1.首先設置尺寸，寬度， 或者高度。width和height優先於size。如果您未進行任何設置，將保留原始視頻分辨率。

2.然後確定在輸入小於目標輸出時是否要增加視頻的大小（高檔）。默認情況下，我們不會升級視頻。通常，您的視頻播放器可以為您放大視頻，因此無需在編碼時放大視頻（這只會增加文件大小）。

3.最後，選擇一種寬高比方法。如果您輸入的視頻的長寬比與您選擇的分辨率不同，則可以保留原始長寬比；也可以保留原始長寬比。拉伸視頻以適合目標寬高比；裁剪輸入以匹配目標寬高比；或用黑條填充視頻以匹配目標寬高比。

注意：許多視頻格式要求寬度和高度值為偶數。如果輸出格式需要這樣做，Zencoder 將在必要時舍入寬度、高度或兩者以滿足此要求。

尺寸

size:String

API版本：V1，V2

上級：輸出

有效值：WxH，其中W和H分別是寬度和高度。兩個尺寸均應為可被4除的整數。

兼容的工作類型：全部

例子：480x320

描述：

輸出文件的分辨率，表示為WxH，例如640×480或1280×720。注意寬度和高度優先於大小。

默認情況下將保留縱橫比模式，因此，如果輸入視頻為16：9，並且要求輸出分辨率為4：3，則輸出將保持16：9縱橫比。這將導致輸出尺寸與要求的尺寸不匹配。用Aspect_mode控制它。

{
  "input": "s3://zencodertesting/test.mov",
  "outputs": [
    {
      "size": "480x320"
    }
  ]
}

高檔

upscale:Boolean

API版本：V1，V2

上級：輸出

默認：錯誤的

有效值：對或錯

兼容的工作類型：全部

例子：真的

描述：

默認情況下，Zencoder不會增加尺寸（或“高檔"）輸入視頻以匹配寬度和高度您在API請求中指定。通常，對視頻進行升頻只會增加大小，但並不能真正提高質量。您的播放設備（Flash，HTML5視頻，iPhone等）可能會增加視頻的大小，而不是讓編碼器來做。

但是，如果確實要強制Zencoder增加輸入視頻的大小，則將“upscale"設置為true。如果這樣做，小於輸出分辨率的輸入視頻將擴展以適合該分辨率。例如，如果您的輸出規格為480×360，並且某人提交的視頻為320×240，那麼如果將此選項設置為true，則視頻將放大為480×360，否則將保持為320×240。

{
  "input": "s3://zencodertesting/test.mov",
  "outputs": [
    {
      "upscale": true
    }
  ]
}

方面 _ 模式

aspect_mode:String

API版本：V1，V2

上級：輸出

默認：保存

有效值：保存，拉伸，裁剪或填充

兼容的工作類型：全部

例子：莊稼

描述：

如果輸入的寬高比與請求的輸出寬高比不匹配，則輸出分辨率應為多少？ “aspect_mode"控制此決定。注意，寬度和高度都可以修改。

例如，假設輸入文件為1280×720（寬屏16：9），但目標輸出為640×480（標準4：3）。解決此問題的方法有四種，Zencoder支持所有這些方法。

  • 保存：默認情況下，Zencoder將保留原始文件的長寬比，因此，如果您提交寬屏內容並要求標準分辨率，則輸出文件將適合保持寬屏長寬比，並且將適合輸出大小。假定玩家隨後將添加黑條以填充框架。在我們的示例中，輸出文件將為640×360（適合640×480目標的寬屏16×9）。
  • 拉緊：當寬高比模式為“拉伸"時，即使輸出視頻使圖像失真，它也會完全符合請求的寬度和高度。因此，在我們的示例中，輸出將為640×480，但看起來是垂直拉伸的。注意：這與上面的“高檔"選項不同，後者用於控制是否將較小的視頻放大到較大的幀大小。
  • 莊稼：此選項告訴Zencoder通過從頂部/底部或左側/右側裁剪像素來“放大"視頻以匹配輸出尺寸。因此，如果輸入為寬屏1280×720，輸出目標為標準640×480，我們將從輸入文件的左側和右側裁剪大約160個像素（創建4：3影片），然後調整其大小降到640×480。如果您的年齡足夠大，可以記住非寬屏VHS（例如我們），則這是舊的“平移和掃描"選項。
  • 軟墊：pad選項告訴Zencoder給信箱添加視頻，以精確匹配輸出幀的大小。如果您的目標播放器不知道如何錄製視頻，請使用此選項。否則，您只是在實際文件中包含黑條而浪費了比特。按照我們的示例，此選項將採用1280×720（16：9）的內容並將其轉換為640×480（4：3），方法是創建640×360的影片，然後在其頂部和底部放置60像素的黑條電影。因此，影片將為640×480，頂部和底部均為黑條。

請注意，aspect_mode要求寬度和一個高度提供（或尺寸）;否則，如果僅提供一個尺寸，我們將按比例計算另一個尺寸。

注意：VP6 video_codec不支持“pad" aspect_mode。如果您要求VP6視頻使用“pad"，我們將使用默認模式“preserve"。

{
  "input": "s3://zencodertesting/test.mov",
  "outputs": [
    {
      "aspect_mode": "crop"
    }
  ]
}

取樣 _ 縱橫比

sample_aspect_ratio:String

API版本：V2

上級：輸出

兼容的工作類型：全部

例子：8：9

描述：

用於輸出視訊的範例 (像素) 外觀比例。大小、寬度、高度和 aspect_ mode 模式選項都是指編碼的像素尺寸，而不是顯示尺寸。

設置用於輸出視頻的樣本/像素的寬高比，指定為“8：9"，“32:27"或任何其他兩個正整數的比率，每個正整數在1到65535之間。

為此設置指定任何（有效）值將默認的 'aspect_mode ' 從' 保存' 到 '伸展'，這意味著默認情況下，您還需要為編碼輸出指定寬度/高度。

筆記：將「高檔」選項與此' 選項搭配使用可預測的輸出解析度可能會很有用。

僅當使用H.264和HEVC編解碼器時，此選項才適用。

{
  "input": "s3://zencodertesting/test.mov",
  "outputs": [
    {
      "sample_aspect_ratio": "8:9"
    }
  ]
}

強制間隔標頭

force_interlace_headers:Boolean

API版本：V2

上級：輸出

兼容的工作類型：全部

默認：錯誤的

描述：

有些解碼器要求交錯的內容，但是可以播放漸進式內容。此設定會在漸進式內容的中繼資料中加入「交錯」標題，使其看起來像是交錯的。

筆記：某些媒體檢查工具不會因此而受到欺騙，儘管有標題信息，它們仍會將該內容報告為漸進式。）

僅當使用H.264編解碼器時，此選項才適用。

{
  "input": "s3://zencodertesting/test.mov",
  "outputs": [
    {
      "force_interlace_headers": true
    }
  ]
}
