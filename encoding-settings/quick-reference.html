編碼設置

本主題提供所有編碼設定的表格，其中包含每個設定詳細指南的連結。

工作

設定 預設 描述
api_key 沒有任何 您的 ZenCoder 帳戶的 API 金鑰。
dynamic_profile_options 沒有任何 使用 CAE 時，每個輸入檔案的輸出數量和參數可能會動態變更。動態設定定義應該建立轉譯的方式。
generate_dynamic_profile 假的 使用 CAE 時，每個輸入檔案的輸出數量和參數可能會動態變更。
input 沒有任何 一個 HTTP，S3，雲文件，GCS，FTP，FTPS，SFTP，Azure，或阿斯佩拉的 URL，我們可以下載文件轉碼。
outputs 沒有任何 輸出設置的數組或散列。
region 我們 處理檔案的區域：美國、歐洲、亞洲或澳洲。
test 假的 啟用工作的測試模式 (「整合模式」)。
private 假的 為工作啟用隱私模式。
download_connections 5 利用多個同時連線進行下載加速 (在某些情況下)。
pass_through 沒有任何 與此工作一起儲存的選擇性資訊。
mock 假的 傳送模擬的工作請求。
grouping 沒有任何 此工作的報表群組。
aspera_transfer_policy 公平 如何配置 Aspera 檔案傳輸的可用頻寬。
transfer_minimum_rate 1000 以 Kbps 為單位的資料傳輸最低限度的目標速率。
transfer_maximum_rate 250000 資料傳輸上限的 Kbps 目標速率。
expected_md5_checksum 沒有任何 輸入檔案的預期總和檢查碼。
credentials 沒有任何 以暱稱參考已儲存的認證。

CAE 設定

設定 預設 描述
min_renditions 1 要產生的轉譯數目下限。
The maximum number of renditions to generate 8 要產生的轉譯數目上限。
min_resolution 192 x 108 階梯要考慮的最低解析度。
max_resolution 來源解析度 階梯要考量的最大解析度。
max_frame_rate 30 可以使用的最大影格速率。
keyframe_rate 0.5 關鍵影格速率。這個值可以是 0.5 或 1。若為啟用 SSAI 的內容，此值必須為 1。
fixed_frame_rate 真的 指示產生器在所有轉譯中使用相同的影格速率。如果為 false，則以低於 360p 解析度的 H.264 基準描述檔轉譯將會被指定為一半的影格速率。
min_bitrate 200 可以使用的最小位元速率 [kbps]。
max_bitrate 15000 可以使用的最大位元速率 [kbps]。
max_first_rendition_bitrate 400 可用於第一次轉譯的最大位元速率 [kbps]。
min_granularity 30 轉譯之間位元速率的最小相對變更。
max_granularity 90 轉譯之間位元速率的最大相對變更。
video_codec h264 要使用的視訊轉碼器。
bitrate_cap_to_bitrate_ratio 1.3 用於 VBR 編碼的最大位元速率與平均位元速率的比率。
buffer_size_to_bitrate_ratio 2.0 用於 VBR 編碼的解碼器緩衝區大小與平均位元率的比率。
bitrate_cap_offset 64 計算最大位元速率時增加常數。
buffer_size_offset 85 計算 VBV 緩衝區大小時增加的常數。
quality_rate_tune 0 品質與速率調整參數。
select_baseline_profile_configuration 真的 縱斷面中使用的至少一個節目將是基準線縱斷面。
video_configurations 沒有任何 描述解析度、影格速率和相關的轉碼器設定的結構陣列，應優先或強制產生為 CAE 編碼的一部分。

格式與轉碼器

設定 預設 描述
format 取決於輸出檔案名稱，然後是視訊或音訊轉碼器。否則：mp4（用於標準輸出）; ts（用於分段輸出）。 要使用的輸出格式。
video_codec 由格式，配置文件或audio_codec。 h264（如果未提供）。 要使用的視訊轉碼器。
audio_codec 由格式，配置文件或video_codec。 AAC（如果未提供）。 要使用的音訊轉碼器。

解析度

設定 預設 描述
size 沒有任何 輸出視訊的解析度 (WxH，以像素為單位)。
width 沒有任何 輸出視訊的最大寬度 (以像素為單位)。
height 沒有任何 輸出視訊的最大高度 (以像素為單位)。
upscale 假的 如果輸入小於目標輸出解析度，則升頻輸出。
aspect_mode 保留 當輸入文件的縱橫比與目標寬度/高度縱橫比不匹配時該怎麼辦。
sample_aspect_ratio 沒有任何 用於輸出視訊的範例 (像素) 外觀比例。大小，寬度，高度和aspect_mode模式選項均指編碼像素尺寸，而不是顯示尺寸。
force_interlace_headers 錯誤的 有些解碼器要求交錯的內容，但是可以播放漸進式內容。此設定會在漸進式內容的中繼資料中加入「交錯」標題，使其看起來像是交錯的。

匯率控制

設定 預設 描述
quality 3 自動選取符合目標視覺品質的最佳視訊位元速率。
video_bitrate 沒有任何 以 kbps 為單位的目標視訊位元速率。如果您選取品質設定，則不需要，除非您要以特定位元速率為目標。
audio_quality 3 自動選擇最佳音訊位元率以符合目標音質。
audio_bitrate 沒有任何 以 kbps 為單位的目標音訊位元率。如果選擇一個，則不需要audio_quality設置，除非您要指定特定的比特率。
max_video_bitrate 沒有任何 最大平均位元速率。
speed 3 目標轉碼速度。較慢的編碼通常允許更高級的壓縮。
decoder_bitrate_cap 沒有任何 提供給解碼器緩衝區的最大位元率。通常用於旨在流式傳輸的視頻或用於定位特定設備的視頻（例如Blu-Ray）。
decoder_buffer_size 沒有任何 解碼器緩衝區的大小，與bitrate_cap。
one_pass 假的 強制單次編碼。
audio_constant_bitrate 假的 如果可能的話，啟用固定位元速率模式。

forced_keyframe_rate 沒有任何 指定每秒的關鍵影格數目，並考慮影格速率 (僅限 H.264)。
generate_frame_index 假的 產生視訊影格索引檔案。
frame_index_filename 沒有任何 在以下情況下覆蓋幀索引的默認文件名generate_frame_index已啟用。
