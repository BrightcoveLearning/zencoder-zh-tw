錯誤代碼

本主題概述了Zencoder API請求的HTTP狀態代碼以及錯誤代碼索引 Zencoder API返回的值。點擊任意錯誤代碼查看更多詳細信息。

HTTP 狀態代碼

下表列出 Zencoder API 要求的 HTTP 回應程式碼 (在發生錯誤時) 會在可能的情況下指出最可能的原因。

HTTP 回應代碼
狀態代碼 可能的說明
200 OK 針對未建立新資產的行動，標準成功回應
201 CREATED 針對建立新資產的要求進行標準回應
204 NO CONTENT 對於沒有響應的請求的標準響應
400 BAD REQUEST 請求主體中的 JSON 無效
401 UNAUTHORIZED 缺少或無效的 API 金鑰
402 PAYMENT REQUIRED 當 API 請求嘗試在尚未設定帳單的帳戶上關閉 Zencoder 整合模式 https://app.zencoder.com/docs/api/accounts/integration 時傳回。你應該去www.zencoder.com然後選擇付款方案
403 FORBIDDEN 速率限制對於超出的請求，或響應對重試次數過多的資源的重試/重新提交API請求
404 NOT FOUND 不正確的請求網址或無效/舊作業ID
406 NOT ACCEPTIBLE 嘗試透過 API 建立非 AWS 憑證而傳回
409 CONFLICT 通常只有在嘗試重新嘗試/重新提交太舊的工作
422 UNPROCESSIBLE ENTITY 當一個 API 請求不能被驗證返回
500 INTERNAL SERVER ERROR ZenCoder 的臨時問題或請求有問題
503 SERVICE UNAVAILABLE Zencoder 暫時無法使用，可能是在定期維護期間
504 GATEWAY TIMEOUT 如果 ZenCoder 服務關閉，則傳回
