概述

我們努力處理盡可能多的檔案，並在第一次成功轉碼，通常建議不要重新提交失敗的檔案。只有下列錯誤的工作才會自動重試，而且我們要求您不要重複重試發生相同錯誤的工作兩次。取而代之聯繫我們帶有工作ID，我們很樂意為您調查文件。從未建議不加選擇地重新提交工作，並這樣做可能會導致限制被放置在您的Zencoder帳戶中。

我們要求您不要重新提交所有失敗的檔案，是因為它會在 ZenCoder 上加載額外的負載，以及它所耗用的資源。我們不會為失敗的工作收費，但是當我們嘗試處理它們時，它們會填補編碼器上的空間，這可能會影響我們盡可能有效地獲得其他工作的能力。通過不重新提交失敗的文件，您可以幫助我們為您降低價格，同時繼續提供快速，高效的編碼。

何時要立即重試

要立即重試的錯誤

如果遇到TranscodingError或一個WorkerTimeoutError，這可能是由於正在處理文件的編碼器出現問題。在這些情況下，再次重試工作並不是不合理的。如果在第二次嘗試時出現相同的錯誤，則可以通過以下方法更好地為您服務與我們聯繫使用工作ID，以便我們進行調查並為您提供有關錯誤原因的更多信息。

何時要稍後重試

由於 Zencoder 增加了對新的轉碼器和檔案的支援，我們有時可以處理先前失敗的檔案。如果任何作業失敗並顯示UnsupportedCodecError或UnsupportedEncodingError，如果添加了支持，它們可能會在以後處理。我們在電子報和 Zencoder 儀表板上宣布新支援的格式，所以如果您發現 Zencoder 先前未支援的轉碼器已新增，搜尋發生這些錯誤的工作將會提供您要重試的檔案清單。

AWS 中斷

在 AWS 中斷的情況下，在某些情況下，工作會自動重試，在某些情況下，您必須重試工作。發生內部問題時 (例如 AWS 中斷、檔案傳輸失敗等)，工作會自動重試。

當工作因為該工作的特定內容（錯誤或無法訪問源文件，錯誤的指定格式等）而失敗時，該工作將失敗，並且將以標準方式通知您。您可以自行重試 (最多 7 天，之後需要開始新的工作)。

檔案傳輸錯誤

如果 ZenCoder 無法成功傳輸檔案，在上傳或下載時，您應該先進行錯誤訊息所建議的任何變更，再重試工作。您可能需要給Zencoder 訪問您的S3存儲桶，確認提供者用戶是否存在（如果在URL中傳遞憑據，授予對該文件的公共訪問權限）或確保網址中的用戶名和密碼已正確轉義。確認檔案的存取權設定正確且 URL 是正確的，請重試該工作。

如果您收到另一個錯誤

任何其他錯誤通常是文件本身的問題-文件可能已損壞，不包含我們可以檢測到的任何媒體或無法處理的方式進行了加密。重新提交這些工作將不太可能導致不同的結果。

如果您遇到某些特定的文件，但除上面所述的錯誤以外，還希望您處理其他錯誤，則應引起我們的注意而不是重新提交。我們會檢閱檔案並解釋為什麼無法處理檔案，或者如果可能的話，將支援新增至 Zencoder。
