快速開始：編碼器

Zencoder的核心是其編碼服務。Zencoder將視頻點播（VOD）內容作為輸入，對其進行編碼，生成具有您指定的格式和其他設置的輸出，並將其交付給您選擇的目的地。在本簡介中，您將學習如何提交基本工作。

資訊編碼器 API

所有操作均使用Zencoder API進行。雖然有一個請求生成器可以用來嘗試一些基本的API請求，您可能很快就會繼續直接訪問API。在本指南中，您可以修改和使用下面顯示的cURL命令，或REST API客戶端，例如郵差。

如果您不熟悉cURL，請參閱設置cURL。

發出請求

提交工作的要求是POST請求發送至：

https://app.zencoder.com/api/v2/jobs

必需的標題

必須與請求一起發送兩個標頭：

  • Zencoder-Api-Key: YOUR_API_KEY
  • Content-Type: application/json

您可以通過訪問獲得API密鑰https://app.zencoder.com/api。在本練習中，您可以使用“完全訪問"或“僅集成"密鑰。

更多資訊

該請求可能（並且通常會包括）作業的輸出設置，包括輸出目的地，通知設置和代碼轉換設置。在這個簡單的練習中，我們將跳過此步驟，但是您可以在編碼設定。

我們目前支持使用以下方式下載文件： HTTP / HTTPS，S3，雲文件，FTP / FTPS，SFTP和Aspera。

一個簡單的新作業請求只需在正文中指定輸入文件的URL，再加上您帳戶的API密鑰在一個標題中Zencoder-Api-Key。

  {
  "input": "s3://zencodertesting/test.mov"
  }

您可以使用以下方法對此進行測試捲曲命令。

  curl --header "Zencoder-Api-Key: YOUR_API_KEY" \
       --header "Content-Type: application/json" \
       --data '{"input":"s3://zencodertesting/test.mov"}' \
       https://app.zencoder.com/api/v2/jobs

將此代碼複製到文本編輯器並替換YOUR_API_KEY與您的API密鑰。然後打開命令提示符，在命令提示符處複製並粘貼cURL命令，然後按Return（回車）鍵。

Windows用戶注意事項：由於Windows上cURL的限制，您需要轉義雙引號，例如\"並包裝--data內容用雙引號代替單引號。

該請求將為該帳戶創建一個編碼作業，並嘗試將文件s3：//zencodertesting/test.mov下載並轉碼到默認輸出目標。

回應

當您通過API創建新的編碼作業時，我們的服務器將立即響應，並提供有關作業和正在創建的輸出文件的詳細信息。您應該存儲作業並輸出ID，以在編碼過程中對其進行跟踪。請注意id是64位範圍內的整數。

數據將以JSON格式返回。

前一個新的編碼作業示例將返回以下內容，並帶有201 Created狀態碼。
