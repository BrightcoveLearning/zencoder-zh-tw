HEVC / H.265概述

HEVC（通常也稱為H.265）可供所有客戶使用。HEVC是下一代視頻編解碼器，與以前的H.264相比，它能夠以更低的比特率提供更高質量的內容。

編碼建議

video_codec_level：限制比特率和編碼樹單元（CTU）。有效值列表。

video_codec_profile：設置編碼配置文件。當前支持的HEVC配置文件為：main，main10，main12，main422-10，main422-12，main444-8，main444-10，main444-12。默認值：main。

video_reference_frames：限制HEVC的參考幀數。預設值：3.

video_bframes：限制用於HEVC的連續bframe的數量。預設值：3.

tuning：調整輸出視頻以針對特定的編碼優先級進行優化。請注意HEVC視頻，允許的值tuning是：

  • 顆粒-針對具有高顆粒水平的膠片進行了優化。
  • psnr-使用“峰值信噪比"來優化視頻質量。
  • ssim-使用“結構相似性"來優化視頻質量。
  • fastdecode-降低編碼複雜度，以便於解碼。
  • 零延遲-x265將保留幀的內部緩衝區以提高質量；此設置將刪除該緩衝區，但會降低質量。

crf：CRF 是位元控制設定，並設定恆定速率因子。Zencoder 品質設定會決定指定視訊的適當 CRF。因此，如果您同時設定 [品質] 和 [CRF]，CRF 設定將會覆寫 [品質]。有效值為 0-51，較低的值則是品質較佳。一般來說，你不應該低於 16; 16 對於大多數文件來說幾乎無損。大約 24 通常看起來相當不錯。大約 35 看起來很壓縮。

  "input": "https://user:pass@bucket-name/file-name.mov",
  "outputs": [
      {
          "label": "hevc",
          "url": "s3://output-bucket/h265.mp4",
          "video_codec": "hevc",
          "video_codec_profile": "main",
          "audio_codec": "aac",
          "format": "mp4",
          "size": "1920x1080"
      },
      {
          "label": "h264",
          "url": "s3://output-bucket/h264.mp4",
          "video_codec": "h264",
          "video_codec_profile": "main",
          "audio_codec": "aac",
          "format": "mp4",
          "size": "640x360"
      }
  ]
}
