常見問題集：基本概念

本主題回答有關Zencoder的一些基本問題。

常見問題

  • 有什麼作用編碼器做？

    編碼器是一種在線視頻轉換/編碼/轉碼服務。我們會從您的網站用戶或其他來源獲取視頻，並將其轉換為與網絡播放，手機或您需要支持的任何其他設備兼容的格式。

  • 誰是編碼器為了？

    編碼器適用於希望將視頻集成到他們的應用程序中而無需經歷構建、維護和擴展他們自己的轉碼服務器的痛苦和緩慢過程的開發人員。任何接受用戶上傳視頻的網站或應用程序都需要某種形式的轉碼後端。用戶上傳的視頻很少具有在網絡和其他設備上可以播放的大小和格式。

  • 如何上傳要編碼的視頻？

    的目的編碼器是自動轉換您的視頻，而無需通過網絡表單手動將每個視頻上傳給我們。為了實現這一點，我們創建了一個應用程序接口允許您的應用程序與編碼器並在需要對新視頻進行編碼時提醒我們。首先，您的視頻應該在文件服務器或云託管服務上，例如 Amazon S3 或 Rackspace Cloud Files，編碼器可以通過網絡訪問。然後你可以使用我們的請求生成器創建一個編碼作業請求，它告訴編碼器在哪裡可以找到您的視頻，如何對其進行編碼，以及將生成的輸出視頻放在哪裡。見請求生成器在您的帳戶中獲取更多詳細信息。

  • 是否可以使用請求生成器？

    是的-使用請求構建器只能上傳小於5GB的文件。

  • 什麼是編碼工作？

    編碼工作是獲取輸入視頻或音頻文件並將其轉換為一種或多種輸出格式的過程。您可以通過原料藥或使用請求生成器在您的帳戶中。在編碼過程中，編碼器首先從託管的任何地方下載輸入文件。然後，我們將文件轉換為指定的輸出設置，然後將輸出文件上傳到我們自己的服務器或您設置的特定位置（通常與輸入文件位於同一託管服務器）。

  • 我可以在一個編碼作業中提交多個輸入嗎？

    不可以，每個作業只能輸入一個。

  • 什麼是編程語言編碼器兼容？

    任何能夠發送HTTP請求的語言。其中包括Python，Ruby，Java，.NET（C＃，VB，F＃等），PHP，Perl，Javascript，Tcl，Haskell，Factor，Common Lisp，Smalltalk，Forth，ML，R，S，J，K， Bash，Clojure，Scala，Modula2，Modula3，ColdFusion，Emacs Lisp，Lua，C，C ++，D，Objective-C，NewtonScript，Mathematica，PL / SQL，Cobol，Arc和Assembly。如果在這裡看不到您喜歡的語言，請檢查它是否支持Internet。如果是這樣，那可能很好。

    換句話說，編碼器 API 只是傳入和傳出 HTTP 請求。我們確實提供了一些集成庫，這使得它更易於使用，但集成起來並不難編碼器沒有圖書館。內置的JSON或XML支持是一個額外的好處，但是JSON非常簡單，如果您碰巧使用了不支持它的語言（Cobol？），您可能可以自己將其砍死。

  • 我需要一個集成庫嗎？

    不。至少，與編碼器只是發送帶有 JSON 或 XML 正文的 HTTP 請求。您可以從帶有curl的命令行。得到201 Created回來，你剛剛與編碼器 .

    除此之外，您可能希望通過HTTP接收output_notification。給我們一個URL，我們將職位狀態回發給您。您只需要在站點上設置一個URL即可接收帶有JSON或XML正文的HTTP POST。

    最後，您可能想使用我們的API來用編碼工作做其他事情，例如working_with_jobs，working_with_jobs或working_with_jobs。那也只是HTTP請求。

  • 集成庫做什麼？

    我們的集成庫將您從原始HTTP請求中抽像出來。使用我們的Ruby庫，例如，您可以創建如下工作：

    Zencoder::Job.create({:input => 's3://bucket/key.mp4',
    :outputs => [{:label => 'vp8 for the web',
    :url => 's3://bucket/key_output.webm'}]})

    而不是做這樣的事情：

    require 'uri'
    require 'json'
    response = Net::HTTP.post_form(URI.parse('jobs.json'),
    {:input => 's3://bucket/key.mp4',
    :outputs => [{:label => 'vp8 for the web',
    :url => 's3://bucket/key_output.webm'}]}.to_json)

    這些庫不支持直接使用API時沒有的任何功能。他們只是使它容易一些。

  • 什麼是整合模式？

    集成模式是您帳戶上的一項設置，可讓您測試您的集成編碼器無需支付編碼費用。您可以在測試作業和集成模式部分。

  • 什麼是隱私模式？

    隱私模式將強制使用某些API參數，以保護您的內容免遭未經授權的觀看。我們還將禁用我們系統拍攝的縮略圖（您仍然可以指定輸出縮略圖）並混淆任何潛在的敏感信息。編碼器員工不會出於任何原因查看私人文件。

    如果您通過以下方式發送成人內容編碼器您的帳戶必須處於隱私模式 .

  • API請求速率受到限制嗎？

    是的，編碼器限制在給定時間範圍內可以調用特定方法的次數。限制是按方法（資源）進行跟踪的，進度請求除外（按輸出跟踪）。例如，取得作業和輸出資訊的呼叫都會計入相同的限制，而輸出進度的呼叫則每個輸出單獨追蹤。

    每個帳戶每分鐘允許 60 個請求，除了工作建立請求 (每分鐘限制為 1,000 個請求) 以及輸出進度請求 (每分鐘限制為每分鐘 60 個請求)。

    每個HTTP響應均包含一個名為“X-Zencoder-Rate-Remaining"。此標頭列出您可以在目前時間範圍內對指定資源進行的呼叫次數。

    如果超出配額，編碼器將返回 403 錯誤，主體為“403 Forbidden (Rate Limit Exceeded)”。我們還將發送回另一個標頭，“Retry-After"，其中包含重置您的配額之前的秒數。

  • 為什麼API請求速率受到限制？

    我們將 API 請求限制為合理數量，以確保一個客戶使用編碼器不會對其他客戶產生不利影響。如果沒有速率限制，一位客戶可能會以阻止其他客戶使用該系統的方式來淹沒我們的系統請求。當客戶第一次使用 API 時，這種情況會偶然發生-一個簡單的編程錯誤可能會淹沒 API，客戶不意味著發送的請求。

    只要您的應用程序設置正確，速率限制應該不會成為問題。當我們在正文中返回帶有「超出價格限制」的 403 時，只需在指定的時間段後重試您的請求即可。

    如果您發現自己經常遇到這些限制，聯繫我們討論您的設置和選項。
