速率限制

API 請求率是否有限？

是的，編碼器限制在給定時間範圍內可以調用特定方法的次數。限制是根據每種方法（資源）進行跟踪的（例如，這意味著，GET和POST請求分別進行跟踪），但進度請求除外（按輸出跟踪）。例如，取得作業和輸出資訊的呼叫都會計入相同的限制，而輸出進度的呼叫則每個輸出單獨追蹤。

每個帳戶每分鐘允許 60 個請求，除了工作建立請求 (每分鐘限制為 1,000 個請求) 以及輸出進度請求 (每分鐘限制為每分鐘 60 個請求)。

每個HTTP響應均包含一個名為“X-Zencoder-Rate-Remaining"。此標頭列出您可以在目前時間範圍內對指定資源進行的呼叫次數。

如果超出配額，Zencoder將返回403錯誤，正文為“403 Forbidden (Rate Limit Exceeded)"。我們還將發送回另一個標頭，“Retry-After"，其中包含重置您的配額之前的秒數。

為什麼 API 請求率受到限制？

我們將 API 請求限制為合理的數量，以確保一位客戶使用 Zencoder 不會對其他客戶造成不利影響。如果沒有速率限制，一位客戶可能會以阻止其他客戶使用該系統的方式來淹沒我們的系統請求。當客戶第一次使用 API 時，這種情況會偶然發生-一個簡單的編程錯誤可能會淹沒 API，客戶不意味著發送的請求。

只要您的應用程序設置正確，速率限制應該不會成為問題。當我們在正文中返回帶有「超出價格限制」的 403 時，只需在指定的時間段後重試您的請求即可。

如果您發現自己經常遇到這些限制，聯繫我們討論您的設置和選項。
