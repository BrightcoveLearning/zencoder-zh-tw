常問問題：檔案傳輸

本主題回答有關將文件傳輸到Zencoder進行編碼以及傳輸到目的地進行輸出的問題。

常見問題

  • 您支持哪些傳輸協議？HTTP？FTP？

    輸入：我們支持HTTP，HTTPS，FTP，SFTP，Rackspace Cloud Files，Google Cloud Storage，Azure和Amazon S3。

    輸出：我們支持FTP，SFTP，Rackspace雲文件，Azure，Google Cloud Storage和Amazon S3。我們目前不支持常規的HTTP上傳。

    雲文件用戶：您可以通過將其添加到協議中來指定要使用的區域（DFW，ORD或UK），例如cf+ord://username:api_key@container/object。當前，僅支持的Cloud Files區域是DFW，ORD和UK。默認情況下將使用DFW區域。

    需要其他轉移方式嗎？讓我們知道

  • 我可以將原始文件放在哪裡進行轉碼？

    我們可以通過HTTP，HTTPS，FTP或SFTP從任何地方提取文件。我們還可以從Azure容器，Google Cloud Storage或S3存儲桶（包括私有存儲桶）和Rackspace Cloud Files中提取文件。

    範例：

    http://example.com/path/to/file.avi
http://s3.amazonaws.com/bucket/file.avi
s3://bucket/file.avi
gcs://bucket/file.avi
cf://username:api_key@container/file.avi
azure://account-name:account-key@container/file.avi
https://example.com/path/to/file.avi
ftp://example.com/path/to/file.avi
sftp://example.com/path/to/file.avi
    關於端口的注意事項：如果需要在文件地址中包含端口，則必須在 URL 中的域末尾聲明，如下所示： 
    sftp://example.com:8888/path/to/file.avi

  • 您可以安全地傳輸文件嗎？

    當然。使用SFTP或HTTPS，並在URL中包含用戶名和密碼，如下所示：

    
https://user:pass@example.com/file.mp4
sftp://user:pass@example.com/file.mp4

  • 您可以從私有S3，Google Cloud Storage或Cloud Files存儲桶中提取文件嗎？

    是的您可以添加訪問權限證書到您的Zencoder帳戶，或為Zencoder授予對存儲桶的讀取訪問權限。

  • 有文件大小限制嗎？

    否。我們沒有文件大小限制，實際上，我們選擇不對大型輸入視頻收取額外費用，以鼓勵客戶向我們發送最好的副本。您發送給我們的視頻質量越高，我們可以為您提供的輸出質量就越高。

    但是，有持續時間限制為24小時源視頻。

  • 完成的視頻去哪兒了？

    您可以選擇將視頻發送到何處。

    我們可以將文件發送到FTP或SFTP服務器，S3或Google Cloud Storage存儲桶或Rackspace Cloud Files。見API文檔中的url選項有關更多信息。

    別忘了添加證書或者使Zencoder可以訪問您的存儲桶如果您使用S3或Google Cloud Storage。

    或者，您可以不指定輸出目標，我們將保留該文件24小時。我們將為您提供可用於下載視頻的URL。我們會在24小時後刪除該視頻，因此請不要忘記下載它。

    從API版本2開始，如果我們無法將文件上傳到您的服務器，我們將把它們上傳到S3並為您提供所有文件的URL。

  • 如果我將編碼的文件發送到S3，如何設置ACL權限？

    使用我們的API。請參閱以下文檔中的API文檔訪問控制有關更多信息。

    • 檔案傳輸

    Zencoder在Amazon EC2實例上運行，因此我們使用多種IP地址進行文件和實時轉碼。亞馬遜定期將新範圍添加到列表中，可以通過AWS找到該範圍IP範圍API。

    通知

    通知事項目前是從sg-77f03012，但這可能隨時更改。為了解決這個問題，我們添加了一個X-Zencoder-Notification-Secret所有已傳遞通知的標頭。可以在原料藥儀表板，您可以在其中查看當前的機密並在必要時生成新的機密。

    此標頭隨每個請求一起提供，可用於保證通知是從您的Zencoder帳戶傳遞來的。筆記：為了使您的通知保密，請確保將通知傳遞到HTTPS端點！

    筆記：服務器可能會移動到新的數據中心，並且IP地址可能會更改，恕不另行通知。

  • 我可以將包含視頻的硬盤運送到Zencoder嗎？

    如果您有大量要編碼的視頻，但又不想通過互聯網傳輸，則可以使用亞馬遜的AWS導入/導出。您將把帶有視頻的硬盤運送到Amazon，後者將為您將它們上傳到S3。

    亞馬遜有簡單說明使您的驅動器準備好發貨，發送到哪裡等等。
